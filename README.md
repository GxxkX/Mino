# Mino

Mino 是一款隐私优先、完全可自部署的个人 AI 助手。它通过实时语音
转录和深度语义分析，将日常语音转化为结构化的数字资产，自动提取
结构化洞察和每日备忘。

本文档提供了关于理解、安装和使用 Mino 的全面指南。无论你是希望
参与贡献的开发者，还是正在搭建个人 AI 助手的终端用户，都可以在
这里找到所需的信息。

## 你可以用 Mino 做什么

Mino 让你能够从任何设备捕获语音笔记，并自动将其转化为有组织的、
可搜索的内容。系统处理从录音、转录到结构化数据提取的整个流程，
同时确保你的数据完全由你掌控。

你可以通过智能手表、手机、网页浏览器或桌面应用录制语音笔记。
每段录音都会被实时自动转录，并由 AI 处理以提取标题、摘要、
待办事项和记忆要点。之后，你可以使用自然语言查询搜索全部历史
记录，或与 AI 助手就过往对话进行交流。

系统支持多设备无缝协作。你可以在手表上开始录音，然后在手机或
电脑上查看结果。所有数据在设备间保持同步，同时完全保留在你自己
的基础设施上。

## 核心功能

Mino 提供了一整套语音笔记捕获和智能检索功能。

**实时语音转录** 从多种输入源捕获音频并以极低延迟进行转录。
系统使用 WebSocket 连接实现低延迟流式传输，并支持在网络不可用
时进行离线录音。

**结构化信息提取** 使用大语言模型分析你的录音，自动生成有意义
的标题、简洁的摘要、可执行的任务列表和重要的记忆要点。
ExtractAgent 处理转录文本并输出结构化的 JSON 数据。

**语义搜索与全文搜索** 结合多种检索策略，帮助你精确找到所需
内容。Milvus 通过使用 LangchainGo 为对话和记忆生成向量嵌入来
提供语义相似性搜索。Typesense 提供快速的全文关键词匹配，作为
互补的检索路径。

**AI 驱动的对话** 让你可以就过往对话提出问题。ChatAgent 使用
检索增强生成（RAG）流水线从你的历史记录中检索相关上下文，并
提供带有来源引用的回答。对话按会话组织，你可以创建、重命名和
删除会话。

**扩展** 允许你为 Mino 添加自定义功能。你可以通过 Web 界面或
API 创建、启用、禁用和配置扩展。

**多设备支持** 确保你可以从任何设备进行录音和查看。支持的客户端
包括智能手表（Flutter）、手机（Flutter）、Web 浏览器（Next.js）
和桌面应用（Go + Wails）。

## 技术栈

Mino 采用现代化、可扩展的架构设计，专为自部署场景打造。

后端使用 Go 1.24 编写，基于 Gin 框架，提供 RESTful API 和
WebSocket 端点以实现实时通信。PostgreSQL 存储结构化数据，包括
用户信息、对话、记忆、任务、聊天会话和扩展。Milvus 存储向量
嵌入，用于对话和记忆的语义相似性搜索。Typesense 提供对话和
记忆的全文搜索能力。MinIO 存储音频文件，Redis 缓存会话数据并
提供速率限制。

AI 能力使用 LangchainGo 进行 LLM 编排和嵌入生成，支持
OpenAI、Zhipu 和 Ollama 等提供商。LangSmith 集成为 LLM 调用
提供可观测性。系统实现了双 Agent 架构：ChatAgent 用于对话交互，
ExtractAgent 用于从转录文本中提取结构化信息。
`EmbeddingProvider` 接口生成向量嵌入，驱动语义搜索流水线。

Web 界面使用 Next.js 14 和 React 18，Zustand 进行状态管理，
Tailwind CSS 进行样式设计。手表应用使用 Flutter 构建，采用
Provider 模式进行状态管理。移动端和桌面端应用遵循类似的架构。

## 默认访问

首次部署 Mino 时，你可以使用默认管理员账户登录。

| 字段 | 值 |
|------|------|
| 用户名 | mino |
| 密码 | admin |
| 角色 | admin |

为确保安全，请在首次登录后立即更改此密码。
